## 4. 硬件差异分析

### 4.1 关键硬件参数对比

| 参数 | Mars X201 | RTX 4090 | 影响 |
|------|-----------|----------|------|
| Warp Size | **64** | 32 | Mars需要更多线程隐藏延迟 |
| L2 Cache | **~2-4MB** | 72MB | ⭐⭐⭐ 关键差异 |
| 理论带宽 | 1843 GB/s | 1008 GB/s | Mars更高 |
| SM数量 | 104 | 128 | - |

### 4.2 L2 Cache大小决定性能上限

```
x向量大小 = numCols × 8 bytes = 1.26M × 8 = 10.8 MB

Mars X201:
- L2 = ~2-4 MB
- 无法缓存x向量
- 每次访问走DRAM → 延迟高
- 需要TPR=8隐藏延迟

RTX 4090:
- L2 = 72 MB
- 完全缓存x向量
- 90%+命中率 → 延迟低
- 1 warp/row足够
```

## 5. 为什么48.7%是Mars上限？

### 5.1 理论分析

| 因素 | 说明 |
|------|------|
| **随机访问模式** | colIdx随机 → 无法利用coalescing |
| **L2无法缓存x向量** | 每次访问走DRAM |
| **内存延迟** | 需要足够线程隐藏延迟 |
| **Warp=64** | 需要TPR=8才能有效隐藏 |

### 5.2 与RTX对比

| 指标 | Mars | RTX | 差距原因 |
|------|------|------|----------|
| 利用率 | 48.7% | 88.8% | L2 Cache大小 |
| 最优配置 | TPR=8 | __ldg | Warp Size不同 |
| 内核时间 | 0.420 ms | 0.425 ms | 理论带宽补偿 |

## 9. 最终优化代码

### 9.1 Mars X201最优配置

```cpp
// Mars X201最优配置 (Warp=64)
const int TPR = 8;   // ⭐⭐⭐ 最关键！
int blockSize = 256;
int gridSize = (numRows * TPR + blockSize - 1) / blockSize;

// Cache配置必须
cudaFuncSetCacheConfig(vector_tpr_kernel<double, 64, 8>, 
                        cudaFuncCachePreferL1);

// Pinned Memory必须
cudaMallocHost(&h_x, numCols * sizeof(double));
```

### 9.2 RTX 4090最优配置

```cpp
// RTX 4090最优配置 (Warp=32)
int blockSize = 256;
int gridSize = (numRows + blockSize / warpSize - 1) / (blockSize / warpSize);

// __ldg提示让数据进入纹理缓存
for (int i = rowStart + lane; i < rowEnd; i += warpSize) {
    int col = __ldg(&colIdx[i]);
    sum += __ldg(&values[i]) * __ldg(&x[col]);
}

// Pinned Memory
cudaMallocHost(&h_x, numCols * sizeof(double));
```

## 10. 经验总结

### 10.1 正确性验证至关重要

- 异常高的性能数据(1335 GB/s)必须验证正确性
- 每个内核单独验证误差，不能只看整体

### 10.2 理解硬件限制

- L2 Cache大小是性能上限的根本因素
- Warp Size决定最优TPR配置

### 10.3 ILP优化在SpMV中无效

- SpMV是内存受限，不是计算受限
- ILP增加寄存器压力反而有害

### 10.4 不同GPU需要不同优化策略

| GPU | 关键优化 | 说明 |
|------|----------|------|
| **Mars X201** | TPR=8 | Warp=64需要更多并行 |
| **RTX 4090** | __ldg | 利用大L2缓存 |
