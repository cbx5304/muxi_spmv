**关键发现**: RCM列重排序对RTX 4090效果显著(+11.4%)，但对Mars X201效果有限(+1.8%)

**原因分析**:
- RTX 4090有72MB L2缓存，可缓存重排序后的热点区域
- Mars X201只有~4MB L2缓存，即使重排序也无法显著提升命中率
- x向量大小: 1.25M × 4B = 5MB，远超Mars L2容量

## 四、硬件瓶颈分析

### 4.1 L2 Cache影响

```
数据访问分析:
- x向量大小: 1.25M × 4B = 5MB
- Mars X201 L2: ~4MB (不足!)
- RTX 4090 L2: 72MB (充裕)

Mars X201问题:
- L2无法缓存完整x向量
- 随机列索引访问导致缓存失效
- 每次访问都需全局内存读取
```

### 4.2 理论性能上限

**Mars X201分析**:
```
理论带宽: 1843 GB/s
实测有效带宽: ~500 GB/s
利用率: 26.6%

无法提高的原因:
1. L2 Cache太小是硬限制
2. 随机访问x向量无法利用缓存
3. 每个元素都需要全局内存访问
```

// 1. Pinned Memory (关键!)
float* h_x;

    const int THREADS_PER_ROW = 8;   // Mars X201

    const int THREADS_PER_ROW = 4;   // RTX 4090

while (idx + THREADS_PER_ROW * 3 < rowEnd) {
    sum0 += values[idx] * __ldg(&x[colIdx[idx]]);
    sum1 += values[idx + THREADS_PER_ROW] * __ldg(&x[colIdx[idx + THREADS_PER_ROW]]);
    sum2 += values[idx + THREADS_PER_ROW * 2] * __ldg(&x[colIdx[idx + THREADS_PER_ROW * 2]]);
    sum3 += values[idx + THREADS_PER_ROW * 3] * __ldg(&x[colIdx[idx + THREADS_PER_ROW * 3]]);
    idx += THREADS_PER_ROW * 4;
}

## 六、结论

### 6.1 Mars X201性能已达硬件极限

- **Kernel利用率**: 26.6%
- **端到端耗时**: 0.84ms
- **根本瓶颈**: L2 Cache太小 (~4MB)

### 6.2 关键发现

1. **Pinned Memory是唯一重大突破**: 端到端+140%
2. **L2 Cache是根本限制**: 无法通过软件优化解决
3. **所有kernel优化效果有限**: +10-17%范围内
4. **共享内存缓存无效**: 矩阵太大无局部性

### 6.3 性能对比

| 指标 | Mars X201 | RTX 4090 | 说明 |
|------|-----------|----------|------|
| Kernel利用率 | 26.6% | 290% | RTX L2缓存优势 |
| Kernel耗时 | 0.34ms | 0.056ms | RTX 6x快 |
| 端到端耗时 | 0.84ms | 1.87ms | Mars传输更快* |

*当前测试中RTX的D2H传输异常慢，需进一步分析
